# UniStock – Setup Instructions

## Prerequisitos
- Tener PHP 8.1+ y Composer instalados.
- Tener una base de datos MySQL/MariaDB configurada.
- Tener una cuenta de Google Cloud para OAuth.

## Paso a paso
1. **Clonar el repositorio**
   ```bash
   git clone https://github.com/your-org/UniStock.git
   cd UniStock
   ```

2. **Instalar dependencias**
   ```bash
   composer install
   npm install && npm run dev   # opcional, si usas assets frontend
   ```

3. **Crear archivo de entorno**
   Copia el ejemplo y edita las variables:
   ```bash
   cp .env.example .env
   ```
   - `APP_URL` debe apuntar a la URL donde correrá la app (e.g., `http://localhost:8000`).
   - `DB_*` configura la conexión a tu base de datos.
   - **Google OAuth**:
     - `GOOGLE_CLIENT_ID` → Obtén el ID de cliente de la consola de Google Cloud.
     - `GOOGLE_CLIENT_SECRET` → Secreto del cliente.
     - `GOOGLE_REDIRECT_URI` → Debe coincidir con la URI de redirección configurada en Google (ej. `http://localhost:8000/auth/google/callback`).
   - **Clave para crear gerentes** (solo si usas roles avanzados):
     - `ADMIN_CREATE_GERENTE_PASSWORD` → Contraseña que permitirá crear usuarios con rol `gerente` o `super_usuario`.

4. **Generar la clave de aplicación**
   ```bash
   php artisan key:generate
   ```

5. **Ejecutar migraciones y seeders**
   ```bash
   php artisan migrate --seed
   ```
   Esto creará las tablas `users`, `proveedores`, etc.

6. **Compilar assets (si usas Vite)**
   ```bash
   npm run build   # para producción
   ```

7. **Iniciar el servidor de desarrollo**
   ```bash
   php artisan serve
   ```
   La app estará disponible en `http://localhost:8000`.

## Verificar login con Google
- Accede a `http://localhost:8000/login` y pulsa **Iniciar sesión con Google**.
- Si ocurre `redirect_uri_mismatch`, revisa que `GOOGLE_REDIRECT_URI` en `.env` coincida exactamente con la URI configurada en la consola de Google (incluye **http/https** y **puerto**).
- Asegúrate de que la URL pública (si usas Ngrok u otro túnel) esté también registrada en la consola.

## Notas adicionales
- El registro ahora obliga a elegir entre **almacenista** o **proveedor**.
- El tema (modo claro/oscuro) se guarda en `localStorage` y persiste entre recargas.
- Si el usuario no usa Google para registrarse, recibirás notificaciones por correo en su bandeja de entrada.
